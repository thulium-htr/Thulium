# Thulium Style Guide

## Branding Guidelines

Thulium uses a strict color palette to convey technical sophistication and reliability.

### Color Palette

*   **Primary Identity**: Deep Purple (`#5A2A82`) - Used for headers, primary actions, and logo marks.
*   **Secondary Details**: Rich Purple (`#7D3CB5`) - Used for links, highlights, and active states.
*   **Background**: Neutral Light (`#F4F4F7`) - Used for page backgrounds to reduce eye strain.
*   **Text**: Near Black (`#121212`) - Used for body text for high contrast.

## Coding Standards

### Python

*   Follow **PEP 8**.
*   Use `snake_case` for functions and variables.
*   Use `CamelCase` for classes.
*   **Docstrings**: All public modules, classes, and functions must have Google-style docstrings.

### Documentation

*   **Tone**: Formal, academic, objective.
*   **Prohibitions**: No emojis, no slang, no first-person singular ("I"). Use "We" or passive voice where appropriate for technical descriptions.
*   **Language**: English (US).
